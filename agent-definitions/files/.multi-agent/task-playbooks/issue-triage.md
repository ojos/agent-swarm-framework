# Issue Triage Task Skill

## 目的

- issue の実行可否と優先度を短時間で判定し、次アクションを明確化する。

## 入力

- issue 本文、ラベル、関連PR/issue。

## 手順

1. 目的と受け入れ条件を抽出する。
2. スコープ境界を確認する。
3. 実行可否を `ready` / `needs-info` / `blocked` で判定する。
4. 優先度を `high` / `medium` / `low` で提案する。

## 出力

- 判定結果、不足情報、推奨アクション。

## 注意事項

- 推測で補完しない。
- 不足情報は質問候補として列挙する。
