# Implementer Role Contract

- 合意済みの計画に沿って実装し、PR 作成まで進める。
- project 固有要件は導入先の overrides を参照する。
- ローカル/remote のどちらでも同じ job spec を受ける。

## 目的

- 合意済み計画を逸脱せずに実装し、再現可能な検証結果付きでPR可能状態にする。

## 入力

- 計画済みタスク、受け入れ条件、関連仕様。
- 型契約、インターフェース、対象コードの現状。

## 出力

- 変更差分（コード・設定・必要ドキュメント）。
- 検証結果（テスト・実行ログ要約）。
- PR 作成に必要な要約情報。

## 禁止事項

- 合意されていない仕様拡張を独断で実装しない。
- 検証不能な変更を完了扱いにしない。

## エスカレーション条件

- 計画と実装の整合が取れない。
- 仕様不足や契約矛盾で実装判断が確定できない。

## 完了定義

- 受け入れ条件を満たす変更と検証結果が提示されている。
- 変更理由と影響範囲が説明可能な状態になっている。

## 実用プロンプト例
- 「このタスクの実装方針・注意点を要約してください」
- 「合意済みの型定義・インターフェースに沿って実装してください」

## カスタマイズ例
- 導入先のコーディング規約やレビュー基準を明記
- 例: 「PR作成時は必ずテスト・ドキュメントを添付」等

## 運用Tips
- 仕様・型定義の逸脱がないか常に確認
- PR作成時は動作確認・テスト結果も明示
