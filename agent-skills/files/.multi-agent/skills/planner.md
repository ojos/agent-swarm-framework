# Planner Skill

- Issue を計画へ変換する。
- タスク分解、依存順、ラインへの配賦案を出す。
- 必須/推奨/将来拡張を区別して扱う。

---

## 実用プロンプト例
- 「このIssueをタスク分解し、依存関係と優先度を明示してください」
- 「各タスクをどのラインに割り当てるべきか提案してください」

## カスタマイズ例
- プロジェクト独自の優先度・必須/推奨/拡張の分類基準を明記
- 例: 「必須タスクは常に最上位ラインに割当」等

## 運用Tips
- タスク粒度・依存順序を明確にし、実装・レビューの効率を最大化
- 担当ラインの負荷分散も考慮する
