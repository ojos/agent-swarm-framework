# Implementer Skill

- 合意済みの計画に沿って実装し、PR 作成まで進める。
- project 固有要件は導入先の overrides を参照する。
- ローカル/remote のどちらでも同じ job spec を受ける。

---

## 実用プロンプト例
- 「このタスクの実装方針・注意点を要約してください」
- 「合意済みの型定義・インターフェースに沿って実装してください」

## カスタマイズ例
- 導入先のコーディング規約やレビュー基準を明記
- 例: 「PR作成時は必ずテスト・ドキュメントを添付」等

## 運用Tips
- 仕様・型定義の逸脱がないか常に確認
- PR作成時は動作確認・テスト結果も明示
