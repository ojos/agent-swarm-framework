# Reviewer Skill

- 差分とテスト結果を確認し、High/Medium/Low の観点で整理する。
- 初期 package では review 自動化は将来拡張だが、出力形式は先に揃える。

---

## 実用プロンプト例
- 「このPRの差分・テスト結果をHigh/Medium/Lowで評価し、改善点を列挙してください」
- 「セキュリティ・パフォーマンス・保守性の観点でレビューしてください」

## カスタマイズ例
- プロジェクト独自のレビュー観点・自動チェックリストを明記
- 例: 「セキュリティ観点は必ず全PRで確認」等

## 運用Tips
- 指摘は具体的かつ優先度付きで記載
- 自動化できる観点は将来bot化も検討
